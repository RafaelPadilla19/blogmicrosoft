Implementaciones predeterminadas en interfaces

Con las publicaciones de la semana pasada que anuncian .NET Core 3.0 Preview 5 y Visual Studio 2019 versión 16.1 Preview 3 , la última característica principal de C # 8.0 ahora está disponible en versión preliminar. Un gran impedimento para la evolución del software ha sido el hecho de que no se podían agregar nuevos miembros a una interfaz pública. Rompería los implementadores existentes de la interfaz; ¡después de todo no tendrían implementación para el nuevo miembro! Las implementaciones predeterminadas ayudan con eso. Ahora se puede especificar un miembro de interfaz con un cuerpo de código, y si una clase o estructura de implementación no proporciona una implementación de ese miembro, no se produce ningún error. En su lugar, se utiliza la implementación predeterminada. Digamos que ofrecemos la siguiente interfaz: 

interfaz ILogger { void Log ( nivel de LogLevel , mensaje de cadena ); }
